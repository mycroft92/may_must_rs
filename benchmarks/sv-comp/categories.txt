# Selection criteria
# ------------------
# We target integer / loop / control-flow programs only:
#   - no floating-point (checker reports UNKNOWN for float IR)
#   - no heap allocation (malloc/new — Step 4 pending)
#   - no concurrency (out of scope)
#   - property: unreach-call (reachability of __VERIFIER_error)
#
# Start with a small slice per category; comment out categories that produce
# too many UNKNOWN results until the relevant feature is implemented.

# Pure integer loop programs — best fit for our loop invariant synthesis.
c/ReachSafety-Loops

# Integer control-flow without loops — exercises acyclic path reasoning.
c/ReachSafety-ControlFlow
